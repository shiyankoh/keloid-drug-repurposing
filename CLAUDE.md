# Keloid Drug Repurposing Pipeline

## What This Is

Personal research pipeline to find FDA-approved drugs that target keloid-relevant molecular pathways. Not a web app — it's a batch data pipeline that outputs a Markdown report.

## Project Structure

```
keloid/
├── scripts/           # Numbered pipeline scripts, run in order
│   ├── 01_seed_keloid_targets.py
│   ├── 02_ingest_drugs.py
│   ├── 03_score_candidates.py
│   └── 04_generate_report.py
├── tests/             # pytest unit tests (transforms + scoring only)
├── data/
│   ├── raw/           # Cached API responses (JSON) — committed
│   ├── keloid.db      # SQLite database — gitignored, regenerated by scripts
│   └── seed_targets.csv
├── output/            # Generated reports — gitignored
├── run_pipeline.sh    # Runs all 4 scripts in sequence
├── TODOS.md           # Deferred work (literature search, clinical trials, safety)
└── CLAUDE.md
```

## Tech Stack

- **Python 3** — scripts + tests
- **SQLite** — local database, no credentials needed
- **pytest** — unit tests
- **APIs:** OpenTargets (GraphQL, no auth), DGIdb (REST, no auth)

## How to Run

```bash
# Full pipeline
./run_pipeline.sh

# Individual steps
python scripts/01_seed_keloid_targets.py
python scripts/02_ingest_drugs.py
python scripts/03_score_candidates.py
python scripts/04_generate_report.py

# Tests
pytest tests/
```

## Key Design Decisions

Read the "Key Design Decisions" table in `keloid-repurposing-pipeline-plan.md` for full rationale. The highlights:

1. **SQLite, not Supabase** — zero-config, no network dependency. Migrate later if needed.
2. **gene_symbol as join key** — DGIdb returns HGNC symbols natively. OpenTargets Ensembl IDs are normalized to HGNC during ingest. Both IDs stored.
3. **Evidence scores:** OpenTargets association scores used directly. Manually seeded targets default to 0.7.
4. **Raw API responses cached** to `data/raw/` as JSON. Scripts can re-run from cache.

## Database Schema

3 tables in SQLite (`data/keloid.db`):

- `keloid_targets` — keloid-associated molecular targets with pathways and evidence scores
- `drugs` — FDA-approved drugs with indications and mechanisms
- `drug_targets` — junction table linking drugs to gene targets

Join key: `gene_symbol` (HGNC standard).

## Critical Edge Cases (must handle)

1. **OpenTargets returns 0 results for keloid** — log warning, don't produce empty report silently
2. **Ensembl ID with no HGNC mapping** — log warning with the unmapped ID, don't silently drop

## Testing Approach

pytest unit tests on transform and scoring logic. Mock API responses using cached JSON fixtures. ~30-40 tests covering:
- Gene symbol normalization (Ensembl → HGNC)
- Drug deduplication by generic_name
- Score calculation (pathway overlap, evidence avg, multi-target bonus)
- Edge cases: zero overlaps, empty API response, tied scores, duplicate gene in same pathway

Do NOT test: Markdown templating, SQLite writes, file I/O.
